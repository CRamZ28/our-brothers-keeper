# Governance

Our Brother's Keeper operates under a stewardship model. The goal is not efficiency of decision-making. The goal is protecting the people this platform serves — and making sure the project stays true to its mission as it grows beyond any one person.

---

## Current State

The project is in its early open-source stage. Caleb Ramsey (founder) currently holds maintainer authority. All major decisions are made by the founder with input from contributors.

As the contributor community grows, governance will evolve. This document reflects the model as it exists now and the principles that will guide its evolution.

---

## The Stewardship Model

OBK is not governed by what is technically interesting, what drives growth, or what any single contributor wants. It is governed by what best serves grieving families.

This means decisions are not always democratic. A pull request with 20 approvals will still be declined if it conflicts with the mission. A contribution from a single developer will be accepted without consensus if it genuinely protects users and aligns with the principles in [PHILOSOPHY.md](PHILOSOPHY.md).

The standard is stewardship — acting as a trustee for the families who will depend on this tool.

---

## Decision Framework

Every significant decision — a new feature, a major architectural change, a policy update — should be evaluated against these four questions:

1. **Does this increase long-term support for grieving families?**
2. **Does this preserve the Primary's control over their space?**
3. **Does this reduce friction without reducing dignity?**
4. **Does this align with the mission as stated in VISION.md and PHILOSOPHY.md?**

A change that answers "yes" to all four should move forward. A change that fails any one of them should not — regardless of technical merit or contributor enthusiasm.

---

## Non-Negotiables

These are not open to discussion or vote. They apply regardless of who is maintaining the project.

- **No selling user data.** The data on this platform belongs to grieving families. It will not be monetized.
- **No advertising.** There will be no ad targeting, no sponsored content, no commercial use of user behavior.
- **No algorithmic manipulation.** The platform will not use engagement-optimization algorithms that prioritize retention over wellbeing.
- **No reduction of Primary control.** The surviving family member controls their space. That authority cannot be reduced by a product decision.
- **No engagement metrics that affect user behavior.** No leaderboards, no supporter rankings, no public post counts.

If this project is ever acquired, forked for commercial use, or integrated into a product that would violate these principles, the open-source license exists precisely to ensure the mission-aligned version can continue independently.

---

## Maintainers

**Current maintainer:** Caleb Ramsey (@cramsey28)

As the contributor base grows, additional maintainers may be added. Maintainers are expected to:

- Understand the mission deeply before reviewing code
- Evaluate PRs against the decision framework above, not just technical quality
- Actively protect the privacy and dignity of the platform's users
- Decline contributions that conflict with the non-negotiables, regardless of external pressure

---

## Contributor Path

There is no formal contributor tier system yet. For now:

- Open issues to discuss significant changes before building them
- Submit pull requests with a clear description of *why* (not just *what*)
- Engage in good faith with the mission, not just the code
- Demonstrate care for the people this platform serves

Maintainers reserve the right to close any PR or issue that conflicts with the mission, without extensive justification.

---

## Conflict Resolution

If there is a disagreement about a decision:

1. Start with the decision framework above
2. Reference [PHILOSOPHY.md](PHILOSOPHY.md) and [VISION.md](VISION.md) for grounding
3. Ask: "who does this serve, and does it serve them well?"
4. Escalate to the maintainer for a final call

The maintainer's decision on mission-critical issues is final. The goal is not consensus — it's faithfulness to the families this exists to protect.

---

## Looking Ahead

As this project matures, governance may evolve toward:

- A small maintainer council with diverse backgrounds (technical, social work, grief support, faith community)
- A formal Code of Conduct
- Defined tiers for contributors and maintainers
- Possibly a nonprofit structure, if that best serves the mission's long-term sustainability

Any evolution in governance will be guided by the same question that guides every other decision: does this make the platform more trustworthy and more useful for grieving families?

If yes, it moves forward. If not, it doesn't.
